// CAPABILITIES

Built for untrusted code execution

Every design decision in VaultRun assumes the agent code is adversarial. Defence-in-depth from image pull to stdout.

🔒
Container isolation
CapDrop ALL, read-only rootfs, /workspace bind-mount, tmpfs /tmp, no-new-privileges. PID limit of 512 to prevent fork bombs.
Docker + seccomp
🛡️
OPA policy enforcement
Write Rego policies to allow or deny any command + args combination per session, org, or actor. Evaluated before every exec — deny by default.
Open Policy Agent
✍️
cosign image verification
Every image is verified against your public key with cosign before it is pulled or run. No signature → no container, no exception.
Sigstore / cosign
📋
Full audit trail
Every session, run, file upload, snapshot, and policy decision is persisted to PostgreSQL with actor, timestamp, and metadata. SIEM export included.
Immutable log
🌐
Egress control
Per-session AllowedHosts list enforced via host-side iptables rules with private-IP rejection. DNS-rebinding protection built in.
iptables + DNS guard
🗝️
Secrets broker
Inject secrets from Vault, AWS Secrets Manager, or env variables. Resolved values are passed to the container at runtime — never stored in the database.
Vault · AWS · Env
⚡
Warm container pool
Pre-start N containers of a given image to eliminate cold-start latency. Each warm container is cosign-verified and seccomp-profiled at creation time.
Sub-100ms sessions
🔑
MCP server
Drop-in MCP (Model Context Protocol) server — connect Claude Code, Claude Desktop, or any MCP host to VaultRun sessions with zero custom tooling.
MCP · Claude Code
🏢
Multi-tenant RBAC
Organisations, members, and roles (viewer / executor / admin). Sessions, runs, audit logs, and artifacts are scoped per-actor with org-level overrides.
Viewer · Executor · Admin
// INSTALLATION

Up and running in minutes

VaultRun ships as a single Docker Compose stack. You need Docker Engine 24+, a host with iptables support, and a PostgreSQL-compatible database.

# 1. Clone the repository
$ git clone https://github.com/nickvd7/vaultrun.git && cd vaultrun

# 2. Create your environment file
$ cp deployments/.env.example deployments/.env
$ nano deployments/.env

# Required variables in .env:
MASTER_API_KEY=your-strong-bootstrap-key
POSTGRES_PASSWORD=your-db-password

# 3. Start the stack
$ docker compose -f deployments/docker-compose.yml up -d

# 4. Verify it's healthy
$ curl http://localhost:8080/health
{"status":"ok","version":"0.1.0"}

# 5. Create your first API key
$ curl -s -X POST http://localhost:8080/api/v1/keys \
   -H "Authorization: Bearer $MASTER_API_KEY" \
   -d '{"name":"my-key"}' | jq .key
"vr_a1b2c3d4e5f6..."
# Download pre-built binary (Linux amd64)
$ curl -Lo vaultrun \
   https://github.com/nickvd7/vaultrun/releases/latest/download/vaultrun-linux-amd64
$ chmod +x vaultrun && mv vaultrun /usr/local/bin/

# Verify the binary
$ vaultrun version
vaultrun v0.1.0 (linux/amd64)

# Run the API server (requires Postgres + Docker)
$ export DATABASE_URL="postgres://vaultrun:pass@localhost/vaultrun"
$ export MASTER_API_KEY="your-bootstrap-key"
$ vaultrun server start
{"level":"INFO","msg":"vaultrun api starting","addr":":8080"}
# Requires Go 1.22+
$ git clone https://github.com/nickvd7/vaultrun.git && cd vaultrun

# Build the API server
$ go build -o bin/vaultrun-api ./cmd/api

# Build the CLI
$ go build -o bin/vaultrun ./cmd/cli

# Run tests
$ go test ./...
ok github.com/nickvd7/vaultrun/internal/auth
ok github.com/nickvd7/vaultrun/internal/docker
ok github.com/nickvd7/vaultrun/internal/runner
...
// QUICK START — STEP BY STEP
01
Prerequisites
Make sure these are available on your host before starting.
# Minimum requirements
Docker Engine 24+ ✔ net_admin capability for iptables egress
PostgreSQL 14+ ✔ or use the bundled docker-compose.yml
Redis 6.2+ optional — for durable async queue
Linux host ✔ iptables + seccomp support required
02
Configure environment
Copy the example env file and set the two required secrets. Everything else has sane defaults.
$ cp deployments/.env.example deployments/.env

# Required
MASTER_API_KEY=$(openssl rand -hex 32)
POSTGRES_PASSWORD=$(openssl rand -hex 16)

# Recommended for production
COSIGN_PUBLIC_KEY=/etc/vaultrun/cosign.pub
OPA_POLICY_FILE=/etc/vaultrun/policy.rego
DB_SSL_MODE=require
METRICS_TOKEN=$(openssl rand -hex 16)
03
Start the stack
Docker Compose brings up the API server, frontend dashboard, PostgreSQL, and Redis in one command.
$ docker compose -f deployments/docker-compose.yml up -d

# Watch the logs
$ docker compose -f deployments/docker-compose.yml logs -f api

# Check health
$ curl http://localhost:8080/health
{"status":"ok"}
04
Create an API key
Use the master key once to mint a regular API key. Store it — it's shown only once.
$ curl -s -X POST http://localhost:8080/api/v1/keys \
   -H "Authorization: Bearer $MASTER_API_KEY" \
   -H "Content-Type: application/json" \
   -d '{"name":"ci-key","expires_in_days":90}'

# Response (save the key — shown once)
{"key": "vr_a1b2c3...", "id": "uuid"}
05
Run your first agent
Create a session, run a command, and inspect the output — all isolated in a sandboxed container.
# Create a session
$ SESSION=$(curl -s -X POST http://localhost:8080/api/v1/sessions \
   -H "Authorization: Bearer $API_KEY" \
   -d '{"image":"python:3.12-slim"}' | jq -r .id)

# Execute a command
$ curl -s -X POST http://localhost:8080/api/v1/sessions/$SESSION/run \
   -H "Authorization: Bearer $API_KEY" \
   -d '{"command":"python","args":["-c","print(42)"]}'

# Result
{"stdout":"42\n", "exit_code":0, "status":"completed"}
// THREAT MODEL

What VaultRun protects against

Each execution layer adds a defence. A compromised agent that breaks one layer still hits the next.

MITIGATED
Container escape via syscalls
Custom seccomp profile blocks ~200 dangerous syscalls (ptrace, mount, unshare, pivot_root). CapDrop ALL removes all Linux capabilities.
MITIGATED
Filesystem tampering
ReadonlyRootfs prevents writes to the container image. Agents can only write to /workspace (bind mount) and /tmp (64 MB tmpfs).
MITIGATED
Malicious image supply chain
cosign signature verification runs before every image pull. Image allowlist (case-insensitive) blocks unapproved registries entirely.
MITIGATED
SSRF / metadata service access
AllowedHosts resolves DNS at creation time and rejects private/link-local IPs. Redirect-following is disabled on all outbound HTTP clients.
MITIGATED
Fork bomb / resource exhaustion
PidsLimit 512 per container, configurable CPU/memory caps, per-actor rate limiting (Redis-backed for multi-instance deployments).
MITIGATED
Secret exposure in logs / DB
Resolved secrets travel in SecretEnv — injected into the container at runtime, never written to the database or audit log.
MITIGATED
Cross-tenant data access
All session and run queries are scoped by actor or org membership. RBAC enforced at every endpoint — always 404 (never 403) to avoid enumeration.
OPERATOR CONFIG
Unrestricted command execution
No built-in command allowlist — deploy an OPA policy file (OPA_POLICY_FILE) to restrict which binaries and arguments agents may run.
